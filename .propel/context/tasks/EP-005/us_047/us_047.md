# User Story - US_047

## Story ID

* ID Format: US_047

## Story Title

* Angular AI Task Creation UI

## Description

* As a **user**, I want an Angular UI screen for AI-powered task creation with a natural language input field, parsed result display, and confirmation controls, so that I can create tasks quickly using natural language.

## Acceptance Criteria

* Given the user is authenticated, When they navigate to the AI task input screen (SCR-011), Then a prominent text input field MUST be displayed for entering natural language instructions.
* Given the NL input field is accessible from the primary navigation, When checking reachability, Then it MUST be accessible in 1 click from any authenticated screen (per UXR-103).
* Given the user submits natural language text, When the AI parses it, Then the parsed result (task title, date/time, category, priority) MUST be displayed with inline accept/reject controls for each field (per UXR-504).
* Given a loading indicator is required, When the AI is processing, Then a skeleton screen or spinner MUST appear within 200ms (per UXR-501).
* Given the AI is unavailable, When the fallback activates, Then the manual task creation form MUST render within 2 seconds without blocking the workflow (per UXR-603).

## Edge Cases

* What happens when the user submits empty text? The system MUST show an inline validation error "Please enter a task description."
* How does the system handle very long input text? The input MUST be truncated to a reasonable limit (e.g., 500 characters) with a character counter.

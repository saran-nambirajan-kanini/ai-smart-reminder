# User Story - US_045

## Story ID

* ID Format: US_045

## Story Title

* Deterministic Fallback When AI Unavailable

## Description

* As a **user**, I want the system to fall back to manual task creation and deterministic processing when the AI service is unavailable, so that I can continue using the application without interruption.

## Acceptance Criteria

* Given the AI service is unavailable (circuit breaker open), When the user accesses the NLP task input, Then the system MUST display the manual task creation form as a fallback within 2 seconds of AI timeout (per UXR-603).
* Given the AI service is unavailable, When task categorization is requested, Then the system MUST skip AI categorization and allow manual category selection.
* Given the AI service is unavailable, When task priority suggestions are requested, Then the system MUST use deterministic sorting (by due date, then manual priority).
* Given the AI service returns low confidence results, When the confidence is below the configured threshold, Then the system MUST present partial results with highlighted uncertain fields.

## Edge Cases

* What happens when AI service becomes available again after being down? The circuit breaker MUST half-open after the configured timeout (30s) and route a test request to verify availability.
* How does the system communicate AI unavailability to the user? A subtle inline notice MUST appear (not a blocking modal) indicating "AI suggestions temporarily unavailable."

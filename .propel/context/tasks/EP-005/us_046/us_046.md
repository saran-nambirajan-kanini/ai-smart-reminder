# User Story - US_046

## Story ID

* ID Format: US_046

## Story Title

* AI Structured Output Schema Validation

## Description

* As a **system**, I want all LLM responses validated against predefined JSON schemas with 95%+ validity rate, so that AI outputs are reliable and safe to process programmatically.

## Acceptance Criteria

* Given an AI request is made with JSON mode, When the LLM returns a response, Then the system MUST validate it against the expected JSON schema before processing.
* Given schema validation passes, When the response is processed, Then the structured data MUST be deserialized into the expected DTO type.
* Given schema validation fails, When the response does not match the expected schema, Then the system MUST log the invalid response, increment a validation failure counter, and fall back to deterministic processing.
* Given the schema validation is monitored, When measured over a rolling window, Then the validity rate MUST be >= 95% (per AIR-Q03).
* Given the validation pipeline exists, When inspecting it, Then each AI feature (NLP parsing, categorization, prioritization, insights) MUST have its own defined JSON schema.

## Edge Cases

* What happens when the LLM returns an empty response? The system MUST treat it as a validation failure and fall back to deterministic processing.
* How does the system handle extra fields in the AI response? The validator MUST ignore unknown fields (lenient deserialization) but require all mandatory fields.

# User Story - US_044

## Story ID

* ID Format: US_044

## Story Title

* AI Task Auto-Categorization

## Description

* As an **authenticated user**, I want the system to auto-suggest a category for my task based on its title and description using AI classification, so that my tasks are organized without manual effort.

## Acceptance Criteria

* Given a user creates or updates a task, When AI categorization is triggered, Then the system MUST suggest a category (Work, Personal, Health, Finance, or user-defined) based on the task title and description.
* Given the AI suggests a category, When presented to the user, Then it MUST be displayed as a suggestion with accept/override controls (not auto-applied).
* Given the user has custom categories, When the AI classifies a task, Then the system MUST consider the user's existing categories in addition to the default categories.
* Given the AI uses few-shot prompted classification, When the LLM processes the request, Then the prompt MUST include examples of each category for consistent classification.

## Edge Cases

* What happens when the task title and description are too vague for classification? The system MUST return "Uncategorized" as the suggestion with low confidence.
* How does the system handle a task that fits multiple categories? The system MUST return the most relevant single category as the primary suggestion.

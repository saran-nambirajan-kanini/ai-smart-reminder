# User Story - US_003

## Story ID

* ID Format: US_003

## Story Title

* PostgreSQL Provisioning and EF Core Setup

## Description

* As a **developer**, I want PostgreSQL 16+ provisioned with Entity Framework Core 8.0 configured for code-first migrations, so that all data persistence operations have a reliable, version-controlled database foundation.

## Acceptance Criteria

* Given the Docker environment is running, When PostgreSQL is provisioned, Then a database instance MUST be accessible on the configured port with the application connection string.
* Given EF Core is configured, When the DbContext is registered, Then it MUST connect to PostgreSQL using Npgsql provider with connection string from application settings.
* Given EF Core is configured, When running `dotnet ef migrations add`, Then a migration file MUST be generated in the Infrastructure project.
* Given a migration exists, When running `dotnet ef database update`, Then the migration MUST apply successfully creating the schema in PostgreSQL.
* Given the database connection is established, When inspecting EF Core configuration, Then all timestamp columns MUST use PostgreSQL `timestamptz` type for UTC storage.

## Edge Cases

* What happens when the database is unreachable at application startup? The application MUST log a descriptive error and fail gracefully with a health check reporting unhealthy status.
* How does the system handle concurrent migration attempts? EF Core migration lock MUST prevent race conditions during schema updates.

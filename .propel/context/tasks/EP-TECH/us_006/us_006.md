# User Story - US_006

## Story ID

* ID Format: US_006

## Story Title

* Health Check Endpoints for Dependencies

## Description

* As an **operations engineer**, I want ASP.NET Core Health Check endpoints that report the status of all external dependencies (database, Redis, AI provider, notification services), so that system health can be monitored and issues detected proactively.

## Acceptance Criteria

* Given the health check middleware is configured, When `GET /health` is called, Then the response MUST return a standardized JSON payload with overall status (Healthy/Degraded/Unhealthy) and individual dependency statuses.
* Given PostgreSQL is running, When the health check executes, Then the database check MUST report Healthy with response time in milliseconds.
* Given Redis is running, When the health check executes, Then the Redis check MUST report Healthy confirming connectivity.
* Given an external dependency is unreachable, When the health check executes, Then that dependency MUST report Unhealthy and the overall status MUST reflect the degraded state.
* Given health checks are configured, When the endpoint is called, Then the response MUST include checks for: database, Redis, and placeholder checks for AI provider and notification services.

## Edge Cases

* What happens when the health check itself times out? Each dependency check MUST have a configurable timeout (default 5 seconds) after which it reports Unhealthy.
* How does the system handle partial degradation? If optional services (Redis) are down but core services (database) are up, overall status MUST be Degraded, not Unhealthy.

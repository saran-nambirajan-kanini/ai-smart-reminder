# User Story - US_029

## Story ID

* ID Format: US_029

## Story Title

* Recurring Reminder Support

## Description

* As an **authenticated user**, I want to set recurring reminders with daily, weekly, monthly, or custom frequency, so that repetitive tasks automatically generate reminders without manual setup each time.

## Acceptance Criteria

* Given a user creates a reminder, When FrequencyType is set to Daily, Then the system MUST calculate NextOccurrence as the next day at the same time after the initial trigger.
* Given a user creates a reminder, When FrequencyType is set to Weekly, Then the system MUST calculate NextOccurrence as 7 days after the initial trigger.
* Given a user creates a reminder, When FrequencyType is set to Monthly, Then the system MUST calculate NextOccurrence as the same day of the next month.
* Given a user creates a reminder, When FrequencyType is set to Custom, Then CustomIntervalDays MUST be provided and the system MUST calculate NextOccurrence as the specified number of days after each trigger.
* Given a recurring reminder is triggered, When the notification is sent, Then the system MUST update LastTriggeredAt and compute the new NextOccurrence.

## Edge Cases

* What happens when a monthly recurrence falls on the 31st and the next month has 30 days? The system MUST use the last day of the month as the fallback date.
* How does the system handle Custom frequency with CustomIntervalDays = 0? The system MUST reject with 400 Bad Request requiring a positive integer.

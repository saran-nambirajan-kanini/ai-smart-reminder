# User Story - US_033

## Story ID

* ID Format: US_033

## Story Title

* Angular Reminder UI Components

## Description

* As a **user**, I want Angular UI for creating/editing reminders (modal) and viewing upcoming reminders, so that I can manage my reminders through an intuitive interface.

## Acceptance Criteria

* Given the user is on a task detail view, When they click "Set Reminder", Then a reminder modal (SCR-009) MUST open with date picker, time picker, recurrence options, and snooze presets.
* Given the reminder modal is displayed, When the user submits a valid reminder, Then the modal MUST close and a success toast MUST appear within 300ms (per UXR-502).
* Given the user navigates to upcoming reminders (SCR-010), When reminders exist, Then they MUST be displayed sorted by time with task title, scheduled time, and frequency indicator.
* Given the upcoming reminders list is empty, When the screen loads, Then an empty state message MUST be displayed with guidance.
* Given any reminder screen is displayed on mobile (320px), When the layout renders, Then it MUST use single-column layout without horizontal scrolling.

## Edge Cases

* What happens when the reminder modal is open and the session expires? The system MUST preserve form data and restore it after re-authentication (per UXR-604).
* How does the system handle the date picker on mobile? The date/time picker MUST be touch-friendly with minimum 44x44px targets (per UXR-205).

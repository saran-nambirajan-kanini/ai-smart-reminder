# User Story - US_028

## Story ID

* ID Format: US_028

## Story Title

* Reminder CRUD API Endpoints

## Description

* As an **authenticated user**, I want to create, update, and delete reminders associated with my tasks, specifying date, time, and optional recurrence, so that I receive timely notifications for important deadlines.

## Acceptance Criteria

* Given an authenticated user has a task, When they create a reminder with a future date/time, Then the system MUST create the reminder linked to the task and return 201 Created.
* Given a reminder exists, When the user updates the time or associated task, Then the system MUST update the reminder and refresh the UpdatedAt timestamp.
* Given a reminder exists, When the user deletes it, Then the system MUST permanently remove the reminder record and cancel any scheduled background jobs.
* Given a reminder creation request, When the scheduled time is in the past, Then the system MUST return 400 Bad Request with "reminder time must be in the future" error.
* Given a reminder is created, When inspecting the database, Then the ScheduledTime MUST be stored in UTC (timestamptz).

## Edge Cases

* What happens when a user creates a reminder for another user's task? The system MUST return 404 Not Found due to row-level isolation.
* How does the system handle a reminder for a completed task? The system MUST allow it (user may want follow-up reminders).

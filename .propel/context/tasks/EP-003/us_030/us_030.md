# User Story - US_030

## Story ID

* ID Format: US_030

## Story Title

* Hangfire Background Job Scheduling for Reminders

## Description

* As a **system**, I want Hangfire configured with PostgreSQL storage to process reminder triggers within a 1-minute accuracy window, so that reminders fire reliably and persist across application restarts.

## Acceptance Criteria

* Given Hangfire is configured, When the application starts, Then Hangfire MUST initialize with PostgreSQL storage and begin processing scheduled jobs.
* Given a reminder is created, When its ScheduledTime arrives, Then Hangfire MUST trigger the reminder processing within a 1-minute accuracy window of the configured time.
* Given Hangfire is processing reminders, When a reminder triggers, Then the system MUST publish a domain event for notification delivery and update the reminder's LastTriggeredAt timestamp.
* Given a recurring reminder triggers, When processing completes, Then Hangfire MUST schedule the next occurrence based on the FrequencyType and CustomIntervalDays.
* Given the application restarts, When Hangfire reinitializes, Then all pending scheduled jobs MUST be recovered from PostgreSQL storage and processed.

## Edge Cases

* What happens when multiple reminders are scheduled for the same second? Hangfire MUST process all of them concurrently without blocking.
* How does the system handle a reminder trigger that fails? Hangfire MUST retry the job up to 3 times with exponential backoff before marking as failed.

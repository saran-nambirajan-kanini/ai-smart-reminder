# User Story - US_031

## Story ID

* ID Format: US_031

## Story Title

* Reminder Snooze Functionality

## Description

* As an **authenticated user**, I want to snooze a triggered reminder by 5, 15, 30, or 60 minutes (or a custom duration), so that I can defer a reminder without dismissing it permanently.

## Acceptance Criteria

* Given a reminder has been triggered, When the user selects a snooze duration (5/15/30/60 minutes), Then the system MUST reschedule the reminder by setting SnoozedUntil to the current time plus the selected duration.
* Given a reminder is snoozed, When the snooze duration elapses, Then the system MUST re-trigger the reminder notification.
* Given a reminder is snoozed, When the user selects a custom duration, Then the system MUST accept any positive integer (in minutes) and reschedule accordingly.
* Given a reminder has not been triggered yet, When the user attempts to snooze it, Then the system MUST return 400 Bad Request with "reminder has not been triggered yet" error.

## Edge Cases

* What happens when a user snoozes a reminder multiple times? Each snooze MUST override the previous SnoozedUntil value with the new duration.
* How does the system handle a custom snooze of 0 minutes? The system MUST reject with 400 Bad Request requiring a positive duration.

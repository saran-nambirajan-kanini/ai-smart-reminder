# User Story - US_070

## Story ID

* ID Format: US_070

## Story Title

* AI PII Sanitization Service

## Description

* As a **system**, I want user inputs sanitized to remove or mask PII before sending to the AI model provider, so that personally identifiable information is not transmitted to external AI services.

## Acceptance Criteria

* Given a user submits text for AI processing, When the input is prepared for the AI request, Then the PII sanitization service MUST remove or mask: email addresses, phone numbers, physical addresses, and names that are not task-relevant.
* Given PII is detected in the input, When sanitization is applied, Then the sanitized version MUST retain enough context for accurate AI processing (e.g., replace "john@example.com" with "[EMAIL]").
* Given the sanitization service runs, When a prompt is sent to the AI provider, Then the original PII MUST NOT appear in the transmitted prompt.
* Given the sanitization is logged, When checking audit records, Then only the sanitized prompt (not the original) MUST be stored in AIInteractionLog.PromptSummary.

## Edge Cases

* What happens when the entire input is PII (e.g., "Call John at 555-1234")? The system MUST sanitize the PII portions while retaining the action verb and intent.
* How does the system handle false positives (e.g., "Email the project report")? The word "Email" in a non-PII context MUST NOT be masked.

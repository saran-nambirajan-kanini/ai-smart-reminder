# User Story - US_072

## Story ID

* ID Format: US_072

## Story Title

* AI Output Filtering and Token Budget Enforcement

## Description

* As a **system**, I want AI-generated outputs filtered for harmful content and token budgets enforced at 4,000 tokens per request, so that AI outputs are safe and API costs are controlled.

## Acceptance Criteria

* Given an AI response is received, When the output filter processes it, Then it MUST check for and block: harmful content, executable code, and prompt leakage before presenting to users (per AIR-S03).
* Given an AI request is prepared, When the token count is estimated, Then the system MUST enforce a maximum of 4,000 tokens (input + output) per request (per AIR-O01).
* Given a request would exceed the token budget, When detected before sending, Then the system MUST truncate the input to fit within budget or reject with a descriptive error.
* Given the token budget is enforced, When the AI response exceeds the output token limit, Then the system MUST truncate the response at a logical boundary.
* Given output filtering detects harmful content, When the content is blocked, Then the incident MUST be logged and the user shown a safe fallback response.

## Edge Cases

* What happens when the AI attempts prompt injection in its response? The output filter MUST detect and strip injection patterns before the response reaches the user.
* How does the system handle token budget differences between models? Token limits MUST be configurable per model (GPT-4o-mini vs GPT-4o).

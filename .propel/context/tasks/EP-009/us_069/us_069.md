# User Story - US_069

## Story ID

* ID Format: US_069

## Story Title

* Circuit Breaker for External Services

## Description

* As a **system**, I want Polly circuit breaker policies on all external service calls (AI, notifications, OAuth) with configurable failure thresholds, so that cascading failures are prevented and the system remains responsive during outages.

## Acceptance Criteria

* Given an external service fails 5 times within 60 seconds, When the threshold is reached, Then the circuit breaker MUST open for 30 seconds, rejecting further requests immediately.
* Given the circuit is open, When the 30-second timeout elapses, Then the circuit MUST enter half-open state and allow a single test request.
* Given the test request succeeds in half-open state, When confirmed, Then the circuit MUST close and resume normal traffic.
* Given circuit breaker policies exist, When inspecting them, Then separate policies MUST be configured for: AI provider, Firebase (push), SendGrid (email), Twilio (SMS), and OAuth providers.
* Given the circuit opens, When the event is logged, Then it MUST include the service name, failure count, and open duration in structured log output.

## Edge Cases

* What happens when the circuit is open and a critical request arrives? The system MUST immediately return the fallback response (e.g., deterministic AI fallback) without waiting.
* How does the system handle circuit breakers across multiple API instances? Each instance MUST maintain its own circuit breaker state (Polly is per-instance by default).

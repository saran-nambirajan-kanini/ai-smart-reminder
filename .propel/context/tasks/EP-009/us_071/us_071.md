# User Story - US_071

## Story ID

* ID Format: US_071

## Story Title

* AI Interaction Audit Logging

## Description

* As a **system**, I want all AI prompts (sanitized) and responses logged to the AIInteractionLog entity with 12-month retention, so that AI usage is auditable for compliance and cost tracking.

## Acceptance Criteria

* Given an AI request is made, When the request completes (success or failure), Then an AIInteractionLog record MUST be created with: UserId, RequestType, PromptSummary (sanitized), ResponseSummary, TokensUsed, LatencyMs, ModelVersion, and CreatedAt.
* Given the AI interaction log is stored, When the PromptSummary is inspected, Then it MUST contain only the PII-sanitized version of the prompt.
* Given AI interaction logs accumulate, When the retention period is checked, Then logs MUST be retained for a minimum of 12 months.
* Given an AI request fails, When the failure is logged, Then the ResponseSummary MUST contain the error type and message.

## Edge Cases

* What happens when logging the AI interaction fails? The AI request MUST still proceed; logging failure MUST NOT block the user's request.
* How does the system handle very large AI responses in the log? ResponseSummary MUST be truncated to a configurable maximum length (e.g., 4000 characters).

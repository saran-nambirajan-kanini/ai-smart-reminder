# User Story - US_073

## Story ID

* ID Format: US_073

## Story Title

* AI Model Version Management

## Description

* As an **admin**, I want to configure AI model versions and roll back within 1 hour via application configuration without code deployment, so that model changes can be managed safely and quickly.

## Acceptance Criteria

* Given the AI model configuration exists, When an admin updates the model version (e.g., from GPT-4o to GPT-4o-mini), Then the change MUST take effect for new requests without application restart.
* Given a model version change causes issues, When an admin initiates rollback, Then the previous model version MUST be restored within 1 hour (per AIR-O03).
* Given model versions are configured, When inspecting the configuration, Then it MUST include: model name, deployment name, API version, and a timestamp of when the configuration was last changed.
* Given a model version change is made, When the change is saved, Then the event MUST be logged in the audit log with before/after values.

## Edge Cases

* What happens when the configured model version is not available on the AI provider? The system MUST validate the model version against the provider API before accepting the configuration change.
* How does the system handle in-flight requests during a model version switch? In-flight requests MUST complete with the old model; new requests use the updated model.

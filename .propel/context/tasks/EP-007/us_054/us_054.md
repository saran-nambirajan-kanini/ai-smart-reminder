# User Story - US_054

## Story ID

* ID Format: US_054

## Story Title

* Background Analytics Aggregation Job

## Description

* As a **system**, I want a Hangfire background job that pre-aggregates analytics data into UserProductivityMetrics every 5 minutes, so that dashboard queries are fast and don't require real-time computation.

## Acceptance Criteria

* Given Hangfire is running, When the analytics aggregation job executes, Then it MUST compute and upsert UserProductivityMetrics records for all active users.
* Given the aggregation job runs, When metrics are computed, Then it MUST calculate: TotalTasks, CompletedTasks, MissedReminders, OnTimeCompletionRate for the current period.
* Given the job is scheduled, When checking its frequency, Then it MUST run at minimum every 5 minutes (per NFR-002).
* Given the aggregation job completes, When the dashboard endpoint reads data, Then it MUST use the pre-aggregated metrics for fast response.

## Edge Cases

* What happens when the aggregation job takes longer than 5 minutes? The next scheduled run MUST wait for the current one to complete (no overlapping executions).
* How does the system handle a user with millions of records? The aggregation query MUST be optimized with B-tree indexes on (UserId, CompletedAt) and process records in batches of 1,000.

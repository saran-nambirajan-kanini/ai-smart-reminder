# User Story - US_053

## Story ID

* ID Format: US_053

## Story Title

* Dashboard Data Aggregation Endpoint

## Description

* As an **authenticated user**, I want a dashboard endpoint that returns aggregated metrics (total tasks, completed, pending, overdue, upcoming reminders), so that I can see my task management status at a glance.

## Acceptance Criteria

* Given an authenticated user calls the dashboard endpoint, When data exists, Then the system MUST return: total tasks, completed tasks, pending tasks, overdue tasks (past due date, not completed), and count of upcoming reminders.
* Given the dashboard data is requested, When inspecting response time, Then the endpoint MUST respond within 3 seconds (per NFR-001) by reading from pre-aggregated data.
* Given dashboard metrics are displayed, When on desktop viewport (1024px+), Then key metrics MUST be visible above the fold without scrolling (per UXR-104).
* Given the dashboard endpoint is called, When the user has no tasks, Then all counters MUST return 0.

## Edge Cases

* What happens when the pre-aggregated data is stale (> 5 minutes)? The system MUST still return the cached data and trigger a background refresh.
* How does the system handle the first-time user with no data? The dashboard MUST display zero metrics with onboarding guidance.

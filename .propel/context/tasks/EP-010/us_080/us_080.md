# User Story - US_080

## Story ID

* ID Format: US_080

## Story Title

* Uptime Monitoring and Alerting Setup

## Description

* As an **operations engineer**, I want uptime monitoring configured with alerting for 99.9% SLA tracking, so that system availability is continuously monitored and incidents are detected quickly.

## Acceptance Criteria

* Given monitoring is configured, When health check endpoints are polled, Then the system MUST check availability at minimum every 1 minute.
* Given the uptime SLA is 99.9%, When downtime is detected, Then an alert MUST be triggered within 2 minutes of the first failed health check.
* Given uptime metrics are collected, When an admin reviews them, Then monthly uptime percentage and incident history MUST be available.
* Given an alert is triggered, When the system recovers, Then a recovery notification MUST be sent confirming the incident is resolved.

## Edge Cases

* What happens when the monitoring system itself has an outage? A secondary monitoring check MUST exist (e.g., external synthetic monitor) to cover this gap.
* How does the system handle scheduled maintenance windows? Maintenance windows MUST be excluded from SLA calculations (per NFR-004).

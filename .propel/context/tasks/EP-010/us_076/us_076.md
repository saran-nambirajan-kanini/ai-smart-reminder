# User Story - US_076

## Story ID

* ID Format: US_076

## Story Title

* Infrastructure Layer Integration Tests

## Description

* As a **developer**, I want integration tests achieving minimum 60% code coverage for the Infrastructure layer, so that database operations, external service integrations, and middleware are verified against real dependencies.

## Acceptance Criteria

* Given the Infrastructure layer has EF Core repositories, When integration tests are run against a test database, Then coverage MUST be >= 60% for the Infrastructure project.
* Given EF Core repositories are tested, When CRUD operations are performed, Then the tests MUST verify actual database behavior (inserts, queries, updates, deletes) including cascade rules and global query filters.
* Given external service clients exist (AI, notification), When tested, Then integration tests MUST verify request/response serialization against mock HTTP endpoints.
* Given the API middleware pipeline is tested, When API integration tests run, Then authentication, rate limiting, and error handling middleware MUST be verified end-to-end.

## Edge Cases

* What happens when the test database is unavailable? Integration tests MUST skip gracefully with a clear message rather than failing with cryptic connection errors.
* How does the system handle test data cleanup? Each integration test MUST use a transaction that is rolled back after the test, or use a fresh database per test class.

# User Story - US_075

## Story ID

* ID Format: US_075

## Story Title

* Domain and Application Layer Unit Tests

## Description

* As a **developer**, I want unit tests achieving minimum 80% code coverage for Domain and Application layers, so that core business logic is verified and regression-protected.

## Acceptance Criteria

* Given the Domain layer has entities and value objects, When unit tests are run, Then coverage MUST be >= 80% for the Domain project.
* Given the Application layer has services and validators, When unit tests are run, Then coverage MUST be >= 80% for the Application project.
* Given domain entities have validation rules, When tested, Then each validation rule MUST have at least one positive and one negative test case.
* Given application services depend on repository interfaces, When tested, Then dependencies MUST be mocked using Moq with verified interactions.
* Given coverage reports are generated, When inspecting them, Then untested code paths MUST be identifiable for targeted test addition.

## Edge Cases

* What happens when a new service is added without tests? The CI pipeline MUST fail if coverage drops below the 80% threshold.
* How does the system handle testing async methods? All async application services MUST have async test methods using `async Task` pattern.

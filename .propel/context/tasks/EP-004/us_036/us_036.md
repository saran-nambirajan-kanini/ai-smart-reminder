# User Story - US_036

## Story ID

* ID Format: US_036

## Story Title

* SMS Notification via Twilio

## Description

* As a **user with SMS notifications enabled**, I want to receive SMS notifications when a reminder triggers, so that I am notified via text message on my mobile device.

## Acceptance Criteria

* Given a reminder triggers and the user has SMS notifications enabled, When the notification service processes the event, Then an SMS MUST be sent via Twilio to the user's registered phone number.
* Given an SMS is sent, When Twilio returns a successful response, Then the Notification record MUST be created with Channel=SMS and DeliveryStatus=Sent.
* Given Twilio provides a delivery status callback, When the callback is received, Then the system MUST update the Notification's DeliveryStatus to Delivered or Failed accordingly.
* Given an SMS is sent, When inspecting its content, Then it MUST contain the task title and reminder time in a concise format suitable for SMS (< 160 characters).

## Edge Cases

* What happens when the user has no phone number registered? The system MUST skip SMS delivery and proceed to other channels.
* How does the system handle Twilio service outage? The notification MUST be queued for retry per the retry policy (US_037).

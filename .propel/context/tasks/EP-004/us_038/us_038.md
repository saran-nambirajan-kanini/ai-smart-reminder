# User Story - US_038

## Story ID

* ID Format: US_038

## Story Title

* Notification Preferences and Quiet Hours

## Description

* As an **authenticated user**, I want to configure my notification preferences per channel (push, email, SMS) and set quiet hours during which no notifications are sent, so that I control when and how I receive notifications.

## Acceptance Criteria

* Given an authenticated user accesses notification settings, When they enable/disable individual channels (push, email, SMS), Then the system MUST save the per-channel preferences.
* Given an authenticated user sets quiet hours (start time, end time), When a reminder triggers during quiet hours, Then the notification MUST be deferred to the end of the quiet period.
* Given quiet hours are configured, When the quiet period ends, Then all deferred notifications MUST be delivered immediately.
* Given a user disables all notification channels, When they save the preference, Then the system MUST warn the user that reminders will only appear in-app and require explicit confirmation.
* Given notification preferences are updated, When the changes are saved, Then the system MUST return 200 OK with the updated preference state.

## Edge Cases

* What happens when quiet hours span midnight (e.g., 22:00-07:00)? The system MUST correctly handle cross-midnight quiet hour ranges.
* How does the system handle a user in a different timezone? Quiet hours MUST be stored relative to the user's configured timezone and evaluated at trigger time.

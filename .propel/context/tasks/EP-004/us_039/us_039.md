# User Story - US_039

## Story ID

* ID Format: US_039

## Story Title

* Notification History and Mark-as-Read

## Description

* As an **authenticated user**, I want to view my notification history with delivery status and mark notifications as read individually or in bulk, so that I can track past notifications and manage my unread count.

## Acceptance Criteria

* Given an authenticated user requests notification history, When notifications exist, Then the system MUST return a paginated list with message content, delivery channel, delivery status, read status, and timestamp.
* Given an authenticated user marks a single notification as read, When the endpoint is called, Then the system MUST set IsRead=true for that notification and return 200 OK.
* Given an authenticated user calls "mark all as read", When the endpoint is called, Then the system MUST set IsRead=true for all unread notifications belonging to that user.
* Given notifications are returned, When sorted, Then they MUST be ordered by creation timestamp descending (most recent first).
* Given a notification belongs to another user, When the current user tries to mark it as read, Then the system MUST return 404 Not Found.

## Edge Cases

* What happens when the user has no notifications? The system MUST return 200 OK with an empty array.
* How does the system handle marking an already-read notification as read? The system MUST return 200 OK without error (idempotent operation).

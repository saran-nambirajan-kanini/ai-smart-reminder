# User Story - US_034

## Story ID

* ID Format: US_034

## Story Title

* Push Notification via Firebase Cloud Messaging

## Description

* As a **user with push notifications enabled**, I want to receive push notifications on my registered devices when a reminder triggers, so that I am alerted in real-time without needing to check the application.

## Acceptance Criteria

* Given a reminder triggers and the user has push notifications enabled, When the notification service processes the event, Then a push notification MUST be sent via Firebase Cloud Messaging to the user's registered devices.
* Given a push notification is sent, When delivery completes, Then the system MUST create a Notification record with Channel=Push, Content, DeliveryStatus, and SentAt timestamp.
* Given FCM returns a successful delivery, When the notification record is updated, Then DeliveryStatus MUST be set to Sent.
* Given FCM returns a failure, When the delivery fails, Then DeliveryStatus MUST be set to Failed and the failure reason logged.
* Given the web application is running, When FCM registration is configured, Then the application MUST register for push notifications with valid web push certificates.

## Edge Cases

* What happens when the user has no registered devices? The system MUST skip push delivery and log a warning, proceeding to other channels.
* How does the system handle FCM service outage? The notification MUST be queued for retry per the retry policy (US_037).

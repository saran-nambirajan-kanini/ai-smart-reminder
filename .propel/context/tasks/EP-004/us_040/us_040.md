# User Story - US_040

## Story ID

* ID Format: US_040

## Story Title

* Angular Notification Module (Settings, History)

## Description

* As a **user**, I want Angular UI screens for notification settings and notification history, so that I can configure my notification preferences and review past notifications through an accessible interface.

## Acceptance Criteria

* Given the user navigates to notification settings (SCR-014), When the screen loads, Then it MUST display toggles for each channel (push, email, SMS), quiet hours time pickers, and a save button.
* Given the user navigates to notification history (SCR-015), When notifications exist, Then they MUST be displayed with message, channel icon, delivery status badge, read/unread indicator, and timestamp.
* Given the notification history is displayed, When the user clicks a notification, Then it MUST be marked as read with a single click (per UXR-105).
* Given the notification history screen, When a "Mark All as Read" button is clicked, Then all visible notifications MUST update to read state immediately.
* Given any notification screen on mobile, When the layout renders, Then it MUST be responsive with no horizontal scrolling.

## Edge Cases

* What happens when the notification settings form has unsaved changes and the user navigates away? The system MUST prompt the user to save or discard changes (per UXR-604).
* How does the system handle an empty notification history? An empty state with guidance text MUST be displayed.

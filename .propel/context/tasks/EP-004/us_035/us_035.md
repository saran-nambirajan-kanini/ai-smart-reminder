# User Story - US_035

## Story ID

* ID Format: US_035

## Story Title

* Email Notification via SendGrid

## Description

* As a **user with email notifications enabled**, I want to receive email notifications when a reminder triggers, so that I am notified via email when I may not be actively using the application.

## Acceptance Criteria

* Given a reminder triggers and the user has email notifications enabled, When the notification service processes the event, Then an email MUST be sent via SendGrid to the user's registered email address.
* Given an email is sent, When SendGrid returns a successful response, Then the Notification record MUST be created with Channel=Email and DeliveryStatus=Sent.
* Given an email delivery fails, When SendGrid returns an error, Then DeliveryStatus MUST be set to Failed with the error reason logged.
* Given the email is sent, When inspecting its content, Then it MUST contain the task title, reminder time, and a link to the task in the application.

## Edge Cases

* What happens when the user's email address is invalid or bounced? The system MUST update DeliveryStatus to Failed and log the bounce reason.
* How does the system handle SendGrid rate limits? The system MUST queue emails and respect SendGrid's rate limits with exponential backoff starting at 1 second, doubling up to a 60-second maximum interval.

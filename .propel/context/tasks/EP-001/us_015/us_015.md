# User Story - US_015

## Story ID

* ID Format: US_015

## Story Title

* JWT Login with Access and Refresh Tokens

## Description

* As a **registered user**, I want to log in with my email and password and receive JWT tokens, so that I can authenticate subsequent API requests securely with short-lived access tokens and refresh capability.

## Acceptance Criteria

* Given a registered user submits valid email and password, When the login endpoint is called, Then the system MUST return a JWT access token (max 15-minute expiry) and a refresh token.
* Given valid credentials are submitted, When the login succeeds, Then the access token MUST contain claims for UserId, Email, and Role.
* Given a refresh token is submitted to the token refresh endpoint, When the refresh token is valid and not revoked, Then the system MUST issue a new access token and a new refresh token (rotation), revoking the old refresh token.
* Given a refresh token is submitted, When the refresh token has been previously used (replay attack), Then the system MUST revoke all refresh tokens for that user and return 401 Unauthorized.
* Given invalid credentials are submitted, When the login endpoint is called, Then the system MUST return 401 Unauthorized with a generic "invalid credentials" message (not revealing whether email or password was wrong).

## Edge Cases

* What happens when a deactivated user attempts to login? The system MUST return 403 Forbidden with "account deactivated" message.
* How does the system handle concurrent login from multiple devices? Each device MUST receive its own refresh token, and revoking one MUST NOT affect others (unless replay detected).

# User Story - US_020

## Story ID

* ID Format: US_020

## Story Title

* Angular Auth Module with UI Screens

## Description

* As a **user**, I want Angular UI screens for login, registration, password reset, and profile management, so that I can interact with the authentication system through a polished, accessible interface.

## Acceptance Criteria

* Given the Angular application is loaded, When the user is not authenticated, Then the app MUST redirect to the login screen (SCR-001) with email and password fields, social login buttons (Google, Microsoft), and links to registration and password reset.
* Given the login screen is displayed, When the user clicks "Register", Then the app MUST navigate to the registration screen (SCR-002) with name, email, password, and confirm password fields with inline validation.
* Given the registration form is submitted with invalid data, When validation fails, Then inline error messages MUST appear below the relevant input fields (per UXR-503).
* Given the user is authenticated, When navigating to the profile screen (SCR-005), Then the app MUST display the user's current name, email, and notification preferences with edit capability.
* Given any auth form is displayed, When inspecting accessibility, Then all form inputs MUST have programmatic label associations via `for`/`id` or `aria-labelledby` attributes (per UXR-203).

## Edge Cases

* What happens when the JWT token expires during a profile update? The app MUST intercept the 401 response, attempt a silent token refresh, and retry the request without user intervention (per UXR-003).
* How does the system handle the password reset screens? The password reset request (SCR-003) MUST show an email field, and the password reset confirm (SCR-004) MUST show new password and confirm password fields.

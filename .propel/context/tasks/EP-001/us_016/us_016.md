# User Story - US_016

## Story ID

* ID Format: US_016

## Story Title

* Logout and Token Revocation

## Description

* As an **authenticated user**, I want to log out of the application and have my tokens revoked, so that my session is securely terminated and cannot be reused.

## Acceptance Criteria

* Given an authenticated user calls the logout endpoint, When the request includes a valid refresh token, Then the system MUST revoke the refresh token by recording the revocation timestamp.
* Given a user has logged out, When the revoked refresh token is submitted for token refresh, Then the system MUST return 401 Unauthorized.
* Given a user logs out, When the access token is still within its expiry, Then the access token MUST be added to a blacklist (Redis-backed) for the remaining TTL.
* Given the logout request is received, When processing completes, Then the system MUST return 200 OK confirming the logout.

## Edge Cases

* What happens when a user calls logout without a valid refresh token? The system MUST return 400 Bad Request with a descriptive error.
* How does the system handle logout when Redis (blacklist store) is unavailable? The system MUST still revoke the refresh token in the database and log a warning about the blacklist failure.

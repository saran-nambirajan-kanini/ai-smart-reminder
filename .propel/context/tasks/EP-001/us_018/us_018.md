# User Story - US_018

## Story ID

* ID Format: US_018

## Story Title

* OAuth 2.0 Social Login (Google/Microsoft)

## Description

* As a **user**, I want to log in using my Google or Microsoft account, so that I can access the application without creating a separate password.

## Acceptance Criteria

* Given a user selects Google login, When the OAuth flow completes successfully, Then the system MUST create a local user account linked to the Google OAuth ID (if first login) and issue JWT tokens.
* Given a user selects Microsoft login, When the OAuth flow completes successfully, Then the system MUST create a local user account linked to the Microsoft OAuth ID (if first login) and issue JWT tokens.
* Given an existing user with email already registered via email/password, When they attempt OAuth login with the same email, Then the system MUST link the OAuth provider to the existing account (not create a duplicate).
* Given a returning OAuth user, When they log in again, Then the system MUST authenticate against the existing linked account and issue new JWT tokens.
* Given the OAuth provider is unavailable, When the user attempts social login, Then the system MUST display an error message and offer email/password login as an alternative.

## Edge Cases

* What happens when the OAuth provider returns an email that differs from the user's registered email? The system MUST create a separate account or prompt the user to link accounts.
* How does the system handle OAuth token expiry during the authentication callback? The system MUST retry the token exchange once and then fail with a descriptive error.

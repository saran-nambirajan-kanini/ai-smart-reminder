# User Story - US_017

## Story ID

* ID Format: US_017

## Story Title

* Password Reset Flow via Email Link

## Description

* As a **user who forgot their password**, I want to request a password reset via email and set a new password using a time-limited link, so that I can regain access to my account securely.

## Acceptance Criteria

* Given a user submits their email to the password reset request endpoint, When the email exists, Then the system MUST generate a time-limited reset token (60-minute expiry) and send a reset link via email.
* Given a user submits their email, When the email does not exist, Then the system MUST return the same success response (preventing email enumeration) but NOT send an email.
* Given a user clicks the reset link, When the token is valid and not expired, Then the system MUST allow the user to set a new password meeting complexity requirements.
* Given a user submits a new password, When the reset token has expired (> 60 minutes), Then the system MUST return 400 Bad Request with "reset link expired" message.
* Given a password reset is completed, When the new password is set, Then all existing refresh tokens for the user MUST be revoked, forcing re-authentication.

## Edge Cases

* What happens when the same reset token is used twice? The system MUST invalidate the token after first use, returning 400 on subsequent attempts.
* How does the system handle password reset when the email service is unavailable? The system MUST return 500 with a generic error and log the email delivery failure for retry.

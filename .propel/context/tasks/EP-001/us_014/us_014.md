# User Story - US_014

## Story ID

* ID Format: US_014

## Story Title

* User Registration with Validation

## Description

* As a **new user**, I want to register an account with my name, email, and password with proper validation, so that I can securely access the AI Smart Reminder application.

## Acceptance Criteria

* Given a user navigates to the registration page, When they submit valid name, email, and password, Then the system MUST create the user account and return a 201 Created response with JWT tokens.
* Given a registration request is submitted, When the email format is invalid, Then the system MUST return a 400 Bad Request with a descriptive validation error message.
* Given a registration request is submitted, When the password does not meet complexity requirements (minimum 8 characters, uppercase, lowercase, number, special character), Then the system MUST return a 400 Bad Request listing all unmet requirements.
* Given a registration request is submitted, When the email is already registered, Then the system MUST return a 409 Conflict with "email already in use" error message.
* Given a valid registration is completed, When the user account is created, Then the password MUST be hashed using bcrypt with work factor >= 12 and MUST NOT be stored in plaintext.

## Edge Cases

* What happens when the registration request body is empty? The system MUST return a 400 Bad Request listing all required fields (name, email, password).
* How does the system handle SQL injection in the email field? Input MUST be sanitized and parameterized queries used, preventing any injection attacks.
* What happens when the name contains only whitespace? The system MUST reject the registration with a validation error.

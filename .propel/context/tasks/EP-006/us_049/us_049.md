# User Story - US_049

## Story ID

* ID Format: US_049

## Story Title

* AI-Suggested Task Priority per Task

## Description

* As an **authenticated user**, I want the system to suggest a priority level for each task based on deadline proximity and description analysis, allowing me to accept or override the suggestion, so that task priorities are informed by AI analysis.

## Acceptance Criteria

* Given a user creates or views a task, When AI priority suggestion is triggered, Then the system MUST return a suggested priority (High/Medium/Low) based on deadline proximity, description analysis, and completion history.
* Given an AI priority suggestion is displayed, When the user accepts it, Then the task priority MUST be updated to the suggested value.
* Given an AI priority suggestion is displayed, When the user overrides it, Then the user's chosen priority MUST be saved and the override MUST be recorded as AI feedback.
* Given the AI suggestion is for a task with no due date, When the suggestion is generated, Then the system MUST base priority primarily on description analysis and default to Medium if insufficient data.

## Edge Cases

* What happens when the AI suggests the same priority as the current one? The system MUST still display the suggestion for transparency but not require action.
* How does the system handle rapid priority suggestions for many tasks? Suggestions MUST be batched in the ranking request (US_048) rather than making individual API calls per task.

# User Story - US_052

## Story ID

* ID Format: US_052

## Story Title

* Angular AI Priority and Recommendations UI

## Description

* As a **user**, I want Angular UI screens for AI priority ranking view and productivity recommendations, so that I can interact with AI-generated insights through an intuitive interface.

## Acceptance Criteria

* Given the user navigates to AI Priority (SCR-012), When tasks are ranked, Then the screen MUST display a ranked task list with priority badges, AI reasoning text, and accept/override controls per task.
* Given the AI priority view is loaded, When the user overrides a ranking, Then the UI MUST update the task position and display a success toast within 300ms.
* Given recommendations are available, When displayed on the dashboard or dedicated section, Then each recommendation MUST show accept/dismiss controls.
* Given a loading state is triggered, When AI data is being fetched, Then a skeleton screen MUST appear within 200ms (per UXR-501).
* Given the AI service is unavailable, When the priority screen loads, Then the system MUST display deterministic sort with a notice "AI rankings temporarily unavailable" (per UXR-603).

## Edge Cases

* What happens when the task list is empty? The AI priority view MUST display an empty state with guidance to create tasks first.
* How does the system handle the priority view on mobile? The layout MUST adapt to single-column with collapsible reasoning text.

# User Story - US_025

## Story ID

* ID Format: US_025

## Story Title

* Task Filtering and Sorting

## Description

* As an **authenticated user**, I want to filter my tasks by status, priority, category, and due date range, and sort by due date, priority, or creation date, so that I can quickly find and organize relevant tasks.

## Acceptance Criteria

* Given an authenticated user requests their task list, When a status filter is applied (Active/Completed), Then only tasks matching the specified status MUST be returned.
* Given an authenticated user requests their task list, When a priority filter is applied (High/Medium/Low), Then only tasks matching the specified priority MUST be returned.
* Given an authenticated user requests their task list, When a category filter is applied, Then only tasks assigned to the specified category MUST be returned.
* Given an authenticated user requests their task list, When a due date range filter is applied, Then only tasks with due dates within the specified range MUST be returned.
* Given an authenticated user requests their task list, When a sort parameter is specified (due_date, priority, created_at) with direction (asc/desc), Then the results MUST be sorted accordingly.
* Given multiple filters are applied simultaneously, When the task list is retrieved, Then all filters MUST be combined with AND logic.

## Edge Cases

* What happens when no tasks match the filter criteria? The system MUST return 200 OK with an empty array, not 404.
* How does the system handle tasks without a due date when sorting by due date? Tasks without a due date MUST appear at the end of ascending sorts and at the beginning of descending sorts.

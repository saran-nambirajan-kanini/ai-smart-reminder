# User Story - US_026

## Story ID

* ID Format: US_026

## Story Title

* Task Keyword Search

## Description

* As an **authenticated user**, I want to search my tasks by keyword matching against title and description fields, so that I can quickly locate specific tasks.

## Acceptance Criteria

* Given an authenticated user submits a search query, When the keyword matches text in task titles, Then those tasks MUST be returned in the results.
* Given an authenticated user submits a search query, When the keyword matches text in task descriptions, Then those tasks MUST be returned in the results.
* Given a search query is submitted, When results are returned, Then the search MUST be case-insensitive.
* Given a search query is submitted, When combined with existing filters (status, priority, category), Then both search and filters MUST be applied together.

## Edge Cases

* What happens when the search query is empty? The system MUST return the full task list (unfiltered by keyword).
* How does the system handle special characters in the search query? The system MUST escape special characters to prevent injection and search literally.

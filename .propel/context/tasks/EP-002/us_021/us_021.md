# User Story - US_021

## Story ID

* ID Format: US_021

## Story Title

* Task CRUD API Endpoints

## Description

* As an **authenticated user**, I want to create, read, update, and soft-delete tasks with title, description, due date, and priority, so that I can manage my work items through a complete lifecycle.

## Acceptance Criteria

* Given an authenticated user submits a valid task, When the create endpoint is called, Then the system MUST create the task with the provided title (required), optional description, due date, and priority (default Medium), returning 201 Created with the task details.
* Given an authenticated user requests a task by ID, When the task belongs to them and is not soft-deleted, Then the system MUST return 200 OK with full task details including associated categories and notes.
* Given an authenticated user submits updated task fields, When the update endpoint is called, Then the system MUST update only the provided fields and refresh the UpdatedAt timestamp.
* Given an authenticated user deletes a task, When the delete endpoint is called, Then the system MUST soft-delete the task (set IsDeleted=true, DeletedAt=now) without physically removing the record.
* Given an authenticated user requests a task belonging to another user, When the endpoint is called, Then the system MUST return 404 Not Found (row-level isolation via global query filters).

## Edge Cases

* What happens when a task is created without a title? The system MUST return 400 Bad Request with "title is required" validation error.
* How does the system handle updating a soft-deleted task? The system MUST return 404 Not Found since soft-deleted tasks are excluded by global query filters.
* What happens when the task ID does not exist? The system MUST return 404 Not Found.

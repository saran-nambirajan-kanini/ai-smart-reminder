# User Story - US_022

## Story ID

* ID Format: US_022

## Story Title

* Task Completion with Timestamp Recording

## Description

* As an **authenticated user**, I want to mark a task as completed and have the completion timestamp automatically recorded, so that I can track when tasks were finished for analytics purposes.

## Acceptance Criteria

* Given an authenticated user has an active task, When they call the complete endpoint, Then the system MUST set Status=Completed and record the CompletionTimestamp as the current UTC time.
* Given a task is marked as completed, When the task details are retrieved, Then the CompletionTimestamp MUST be included in the response.
* Given a completed task exists, When the user attempts to complete it again, Then the system MUST return 400 Bad Request with "task already completed" error.
* Given a task is completed, When analytics are calculated, Then the completion event MUST be available for productivity metrics computation.

## Edge Cases

* What happens when a user tries to complete a soft-deleted task? The system MUST return 404 Not Found.
* How does the system handle completing a task and then updating it? Completed tasks SHOULD still allow field updates (e.g., adding notes) but the status MUST remain Completed.

# User Story - US_023

## Story ID

* ID Format: US_023

## Story Title

* Category Management and Task Assignment

## Description

* As an **authenticated user**, I want to create, update, and delete custom categories and assign them to tasks (many-to-many), so that I can organize my tasks into meaningful groups.

## Acceptance Criteria

* Given an authenticated user, When they create a category with a name, Then the system MUST create the category and return 201 Created with the category details.
* Given an authenticated user, When they assign one or more categories to a task, Then the system MUST create TaskCategory join records linking the task to the categories.
* Given an authenticated user, When they remove a category assignment from a task, Then the TaskCategory join record MUST be deleted without affecting the task or category.
* Given an authenticated user creates a category with a name that already exists for them, When the create endpoint is called, Then the system MUST return 409 Conflict with "category name already exists" error.
* Given an authenticated user deletes a category, When the delete endpoint is called, Then the category and all its TaskCategory associations MUST be removed.

## Edge Cases

* What happens when a user tries to assign a category belonging to another user? The system MUST return 404 Not Found due to row-level isolation.
* How does the system handle a category name with leading/trailing whitespace? The system MUST trim whitespace before checking uniqueness and storing.

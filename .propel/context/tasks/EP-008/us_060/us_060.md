# User Story - US_060

## Story ID

* ID Format: US_060

## Story Title

* System Health Dashboard API

## Description

* As an **admin**, I want a system health dashboard endpoint showing API response times, error rates, active users, and background job status, so that I can monitor system operational health.

## Acceptance Criteria

* Given an authenticated admin calls the health dashboard endpoint, When the system is operational, Then it MUST return: average API response time, error rate percentage, active user count, and Hangfire job status (queued, processing, succeeded, failed).
* Given health metrics are returned, When inspecting response times, Then the system MUST provide p50, p95, and p99 latency values.
* Given background job status is returned, When inspecting it, Then it MUST include counts for each job state and any failed jobs with error details.
* Given the health endpoint is called, When external dependencies are checked, Then it MUST include status for database, Redis, AI provider, and notification services.

## Edge Cases

* What happens when metrics collection is temporarily unavailable? The system MUST return partial data with a warning indicating which metrics are unavailable.
* How does the system handle very high active user counts? Counts MUST be approximate (using Redis counters) rather than exact to avoid performance impact.

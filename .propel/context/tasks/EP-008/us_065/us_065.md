# User Story - US_065

## Story ID

* ID Format: US_065

## Story Title

* Angular Admin Feature Module

## Description

* As an **admin**, I want Angular UI screens for admin dashboard, user management, audit log, and system settings, so that I can manage the system through an accessible admin interface.

## Acceptance Criteria

* Given an admin user is authenticated, When they navigate to the admin section, Then the admin dashboard (SCR-016) MUST display system health metrics, active user count, and background job status.
* Given the admin navigates to user management (SCR-017), When the screen loads, Then it MUST display a searchable, paginated user table with activate/deactivate actions.
* Given the admin navigates to audit log (SCR-018), When the screen loads, Then it MUST display filterable, paginated audit entries with action, user, entity, and timestamp columns.
* Given the admin navigates to system settings (SCR-019), When the screen loads, Then it MUST display grouped configuration fields (notification, AI, rate limiting) with inline validation and save capability.
* Given a non-admin user, When they attempt to access admin routes, Then the Angular router MUST redirect to 403 Forbidden page.

## Edge Cases

* What happens when admin screens are accessed on mobile? The layout MUST be responsive with table horizontal scrolling for data-heavy views.
* How does the system handle unsaved settings changes? A confirmation dialog MUST appear when navigating away with unsaved changes (per UXR-604).

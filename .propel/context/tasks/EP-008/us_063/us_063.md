# User Story - US_063

## Story ID

* ID Format: US_063

## Story Title

* Data Retention Policy Enforcement

## Description

* As a **system**, I want data retention policies enforced automatically (soft-deleted tasks retained 12 months, audit logs retained 24 months), so that compliance requirements are met without manual intervention.

## Acceptance Criteria

* Given soft-deleted tasks exist, When the retention check runs, Then tasks deleted more than 12 months ago MUST be eligible for permanent deletion.
* Given audit log entries exist, When the retention period is checked, Then entries MUST NOT be modifiable or deletable during the 24-month retention period.
* Given the retention enforcement job runs, When expired records are identified, Then permanent deletion MUST be performed via a background job, not inline API calls.
* Given the retention policies are active, When an admin queries retention status, Then the system MUST report the count of records approaching retention expiry.

## Edge Cases

* What happens when permanent deletion of a soft-deleted task would violate FK constraints? The system MUST cascade-delete dependent records (notes, reminders, notifications) for the expired task.
* How does the system handle audit log entries exactly at the 24-month boundary? The boundary MUST be inclusive (entries at exactly 24 months are still retained).

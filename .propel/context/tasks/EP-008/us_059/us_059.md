# User Story - US_059

## Story ID

* ID Format: US_059

## Story Title

* Admin User Management API

## Description

* As an **admin**, I want to view, search, activate, and deactivate user accounts, so that I can manage the user base and handle account issues.

## Acceptance Criteria

* Given an authenticated admin, When they request the user list, Then the system MUST return a paginated list of all user accounts with name, email, role, active status, and creation date.
* Given an authenticated admin, When they search users by name or email, Then the system MUST return matching results with case-insensitive search.
* Given an authenticated admin, When they deactivate a user account, Then the system MUST set IsActive=false and the user MUST be unable to log in.
* Given an authenticated admin, When they reactivate a deactivated account, Then IsActive MUST be set to true and the user MUST be able to log in again.
* Given an admin attempts to deactivate their own account, When the endpoint is called, Then the system MUST return 400 Bad Request with "cannot deactivate own account" error.

## Edge Cases

* What happens when a non-admin user calls admin endpoints? The system MUST return 403 Forbidden.
* How does the system handle deactivating a user with active sessions? All refresh tokens for the deactivated user MUST be revoked immediately.

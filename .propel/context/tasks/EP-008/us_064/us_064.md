# User Story - US_064

## Story ID

* ID Format: US_064

## Story Title

* Database Backup and Recovery Configuration

## Description

* As an **admin**, I want automated daily database backups with point-in-time recovery within a 30-day window and 4-hour RTO, so that data can be restored in case of failure or data loss.

## Acceptance Criteria

* Given the backup configuration is in place, When a daily backup is triggered, Then the system MUST create a full database backup and store it in the configured backup location.
* Given backups are retained, When checking the backup history, Then backups from the last 30 days MUST be available for restoration.
* Given a point-in-time recovery is requested, When the admin specifies a timestamp within the 30-day window, Then the system MUST be able to restore the database to that exact point in time.
* Given a recovery is initiated, When measuring restoration time, Then the RTO MUST be 4 hours or less (per DR-014).
* Given backups are executed, When the backup completes, Then the system MUST log the backup status, size, and duration.

## Edge Cases

* What happens when a daily backup fails? The system MUST alert admins via notification and retry the backup within 1 hour.
* How does the system handle backup storage capacity? Old backups beyond the 30-day window MUST be automatically purged.

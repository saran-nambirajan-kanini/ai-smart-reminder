# User Story - US_061

## Story ID

* ID Format: US_061

## Story Title

* Audit Log Query API

## Description

* As an **admin**, I want to query audit logs tracking user actions (login, task creation, deletion, settings changes) with filtering and pagination, so that I can investigate user activity for compliance and security purposes.

## Acceptance Criteria

* Given an authenticated admin, When they query the audit log, Then the system MUST return a paginated list of audit entries with: actor, action type, target entity, details, IP address, and UTC timestamp.
* Given the audit log query supports filtering, When filters are applied (by user, action type, date range, entity type), Then only matching entries MUST be returned.
* Given audit log entries exist, When sorted, Then they MUST be ordered by timestamp descending (most recent first) by default.
* Given the audit log is immutable, When any attempt is made to modify or delete entries via API, Then the system MUST return 405 Method Not Allowed.

## Edge Cases

* What happens when the audit log is very large (millions of entries)? The query MUST use indexed columns and pagination to prevent timeout.
* How does the system handle audit log entries for deleted users? Entries MUST be retained with the original UserId; user deletion MUST NOT cascade to audit logs.

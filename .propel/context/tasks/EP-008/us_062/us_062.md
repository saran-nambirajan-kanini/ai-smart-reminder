# User Story - US_062

## Story ID

* ID Format: US_062

## Story Title

* System Settings Configuration API

## Description

* As an **admin**, I want to configure system-wide settings including notification service parameters, AI API configuration, and rate limiting thresholds, so that I can tune system behavior without code deployment.

## Acceptance Criteria

* Given an authenticated admin, When they retrieve current settings, Then the system MUST return all configurable parameters grouped by category (notification, AI, rate limiting).
* Given an admin updates settings, When the update is submitted, Then the system MUST validate the new configuration before saving (e.g., valid API endpoint format, positive rate limit values).
* Given settings are updated, When the change is saved, Then the system MUST log the configuration change in the audit log with before/after values.
* Given AI API configuration is updated, When the new config is validated, Then the system MUST test connectivity to the new endpoint before accepting the change.

## Edge Cases

* What happens when an admin submits an invalid configuration? The system MUST return 400 Bad Request with specific validation errors per field.
* How does the system handle configuration changes during active requests? Active requests MUST complete with the old configuration; new requests MUST use the updated configuration.

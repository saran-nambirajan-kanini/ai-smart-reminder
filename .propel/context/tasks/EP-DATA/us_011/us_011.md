# User Story - US_011

## Story ID

* ID Format: US_011

## Story Title

* Relationship Mappings and Global Query Filters

## Description

* As a **developer**, I want all entity relationship mappings with FK constraints, cascade rules, global query filters for soft-delete and row-level isolation, and unique constraints, so that data integrity is enforced automatically at the database level.

## Acceptance Criteria

* Given all entities are defined, When EF Core relationship mappings are configured, Then FK constraints MUST exist for: Userâ†’Task, Taskâ†’Reminder, Taskâ†’Note, Userâ†’Reminder, Reminderâ†’Notification, Userâ†’Notification, Userâ†’Category, Taskâ†”Category (M:N via TaskCategory), Userâ†’AuditLog, Userâ†’RefreshToken, Userâ†’UserProductivityMetrics, Userâ†’AIInteractionLog.
* Given cascade rules are configured, When a User is soft-deleted, Then associated tasks MUST NOT be cascade-deleted (restrict or set null behavior).
* Given soft-delete global query filter is configured, When querying Tasks through the standard DbSet, Then entities where IsDeleted=true MUST be automatically excluded from results.
* Given row-level isolation filter is configured, When a User queries their tasks, Then the query filter MUST automatically apply UserId-based filtering ensuring users only see their own data.
* Given unique constraints are configured, When a duplicate email is inserted for User or duplicate category name per user is inserted, Then the database MUST reject the insert with a constraint violation.

## Edge Cases

* What happens when an Admin needs to query all users' data? Admin role MUST bypass the row-level isolation global query filter using `IgnoreQueryFilters()`.
* How does the system handle orphaned TaskCategory records when a Category is deleted? Cascade delete MUST remove TaskCategory join records when a Category is deleted.

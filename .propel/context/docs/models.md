#### UC-001: User Registration and Authentication

**Source**: [spec.md#UC-001](.propel/context/docs/spec.md#UC-001)

```mermaid
sequenceDiagram

    participant API as API Controller

    participant Redis as Redis Cache

    Note over User,OAuth: UC-001 - User Registration and Authentication

    alt Email/Password Registration
        User->>SPA: Enter name, email, password
        SPA->>API: POST /api/v1/auth/register
        API->>Auth: ValidateAndRegister(dto)
        Auth->>Auth: Validate email format
        Auth->>Auth: Validate password complexity
        Auth->>DB: Check email uniqueness
        DB-->>Auth: Email available
        Auth->>Auth: Hash password (bcrypt)
        Auth->>DB: Insert User record
        DB-->>Auth: User created
        Auth->>Auth: Generate JWT + Refresh Token
        Auth->>DB: Store RefreshToken
        Auth->>Redis: Cache user session
        Auth-->>API: TokenResponse
        API-->>SPA: 201 Created + tokens
        SPA-->>User: Redirect to dashboard
    else OAuth Login (Google/Microsoft)
        User->>SPA: Click OAuth provider button
        SPA->>OAuth: Redirect to provider
        OAuth-->>User: Login prompt
        User->>OAuth: Authenticate
        OAuth-->>SPA: Authorization code
        SPA->>API: POST /api/v1/auth/oauth
        API->>Auth: ProcessOAuth(code, provider)
        Auth->>OAuth: Exchange code for token
        OAuth-->>Auth: User info + token
        Auth->>DB: Find or create user
        DB-->>Auth: User record
        Auth->>Auth: Generate JWT + Refresh Token
        Auth->>DB: Store RefreshToken
        Auth-->>API: TokenResponse
        API-->>SPA: 200 OK + tokens
        SPA-->>User: Redirect to dashboard
    else Validation Failure
        User->>SPA: Enter invalid data
        SPA->>API: POST /api/v1/auth/register
        API->>Auth: ValidateAndRegister(dto)
        Auth-->>API: ValidationException
        API-->>SPA: 400 Bad Request + errors

    end
```

```mermaid
    Hangfire->>ReminderSvc: TriggerReminder(id)
    ReminderSvc->>DB: Check user quiet hours
    DB-->>ReminderSvc: User preferences

        ReminderSvc->>NotifSvc: SendNotification(reminder, channels)
        NotifSvc->>DB: Create Notification record (Pending)

        par Push Notification
            NotifSvc->>FCM: Send push message
            FCM-->>NotifSvc: Delivery status
        and Email Notification
            NotifSvc->>Email: Send email
            Email-->>NotifSvc: Delivery status

            ReminderSvc->>ReminderSvc: Calculate next occurrence
            ReminderSvc->>DB: Update NextOccurrence
            ReminderSvc->>Hangfire: Schedule next job
        end
    else In Quiet Hours
        ReminderSvc->>Hangfire: Reschedule to end of quiet hours
    end

    opt User Snoozes
        User->>SPA: Snooze (15 minutes)

        API->>ReminderSvc: SnoozeReminder(id, duration)
```
